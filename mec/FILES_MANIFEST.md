## 💾 БАЗЫ ДАННЫХ

### `cable_params.json`
- **Тип:** JSON файл
- **Размер:** ~5-50 KB (растёт при использовании)
- **Создан:** Автоматически при первом расчёте
- **Содержит:** Кэш результатов расчётов (α, β, Z₀)
- **Пример:**
  ```json
  {
    "ШВВП 2×1.5_copper_24MHz": {
      "alpha": 144.19,
      "beta": 1087.28,
      "Z0": 66.85,
      "frequency_mhz": 24.0
    }
  }
  ```
- **Назначение:** Ускорение — не пересчитывать одни и те же параметры
- **Можно ли удалить?** ДА — пересчитается автоматически

### `test_scheme.json`
- **Тип:** JSON файл (пример)
- **Размер:** ~2 KB
- **Создан:** Во время тестирования (`test_cable_save.py`)
- **Содержит:** Пример сохранённой схемы (2 узла + 1 кабель)
- **Назначение:** Показать формат сохранения
- **Можно ли удалить?** ДА — пересоздастся при новом тестировании

## 🧪 ТЕСТЫ

### `test_cable_save.py`
- **Тип:** Python тестовый скрипт
- **Размер:** 150+ строк
- **Запуск:** `python test_cable_save.py`
- **Что тестирует:**
  - Добавление кабеля с типом и материалом
  - Сохранение схемы в JSON
  - Загрузку схемы из JSON
  - Восстановление типа и материала после загрузки
  - Файловые операции (write/read)
- **Результаты:** ✓ ВСЕ ТЕСТЫ ПРОЙДЕНЫ
- **Важно:** Подтверждает, что кабель_type и material сохраняются корректно

### `test_unified.py`
- **Тип:** Python тестовый скрипт
- **Назначение:** Тестирование unified_app.py
- **Размер:** ~100 строк
- **Может потребоваться:** Для дополнительного тестирования
