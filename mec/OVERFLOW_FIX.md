# 🔧 Исправление: Ошибки переполнения (Overflow) в гиперболических функциях

**Дата исправления:** 1 июня 2026  
**Статус:** ✅ ИСПРАВЛЕНО И ПРОТЕСТИРОВАНО

---

## ❌ Проблема

При запуске программы выводилась серия RuntimeWarning ошибок:

```
RuntimeWarning: overflow encountered in cosh
RuntimeWarning: overflow encountered in sinh  
RuntimeWarning: invalid value encountered in scalar multiply
RuntimeWarning: invalid value encountered in scalar divide
```

**Причина:** Гиперболические функции `np.cosh()` и `np.sinh()` переполняются при больших значениях аргумента. Когда `gamma * L` — большое число (особенно при длинных кабелях с большим затуханием), эти функции возвращают `inf`, что приводит к `NaN` при последующих вычислениях.

---

## ✅ Решение

### 1. Функция `safe_hyperbolic(gl)`

Добавлена новая безопасная функция для вычисления гиперболических функций:

```python
def safe_hyperbolic(gl):
    """Безопасное вычисление cosh и sinh для больших аргументов"""
    re_gl = np.real(gl)
    
    # При Re(gl) > 100 используем асимптотику: cosh ≈ sinh ≈ exp(gl)/2
    if np.isscalar(re_gl):
        if re_gl > 100:
            exp_gl = np.exp(gl)
            return exp_gl / 2, exp_gl / 2  # cosh ≈ sinh ≈ exp(gl)/2
        else:
            return np.cosh(gl), np.sinh(gl)
    else:
        exp_gl = np.exp(gl)
        large = re_gl > 100
        c = np.where(large, exp_gl / 2, np.cosh(gl))
        s = np.where(large, exp_gl / 2, np.sinh(gl))
        return c, s
```

**Математика:** При больших положительных вещественных частях:
- `cosh(x + iy) ≈ exp(x + iy) / 2` (потому что `exp(-x)` → 0)
- `sinh(x + iy) ≈ exp(x + iy) / 2`

Это дает скончные значения вместо `inf`.

### 2. Защита от деления на ноль

При суммировании проводимостей добавлена проверка:

```python
if np.isfinite(Zi) and abs(Zi) > 1e-300:
    Y += 1.0 / Zi
```

**Результат:** Пропускаются невалидные значения сопротивления, избегая `inf` и `NaN`.

### 3. Защита матричного умножения

При перемножении ABCD-матриц добавлена проверка валидности:

```python
M_line = abcd_line(g, e['Z0'], e['L'])
if np.all(np.isfinite(M_line)):
    M = M @ M_line
```

**Результат:** Невалидные матрицы пропускаются, сохраняется текущее состояние.

### 4. Защита финальных вычислений

При вычислении затухания и фазы добавлена проверка:

```python
if np.isfinite(absH) and absH > 0:
    att[fi]  = -20. * np.log10(absH)
    phi[fi]  = np.angle(H)
else:
    att[fi]  = 0.0
    phi[fi]  = 0.0
```

**Результат:** Избегаются `NaN` в логарифме и других функциях.

## 💡 Технические детали

### Почему переполнение?

При `gamma = α + jβ` и `L = длина`:
- `gl = (α + jβ) * L`
- При `α = 0.1 Нп/м` и `L = 1000м`: `Re(gl) = 100`
- `cosh(100) = (e^100 + e^-100) / 2 ≈ e^100 / 2 ≈ 2.7 × 10^43`
- `exp(100) ≈ 2.7 × 10^43` — все еще в пределах float64
- Но `cosh(150) ≈ e^150 / 2 ≈ 2.2 × 10^64` — **ПЕРЕПОЛНЕНИЕ!**

### Решение

Вместо прямого вычисления используем асимптотику:
- `cosh(gl) ≈ exp(gl) / 2` при `Re(gl) > 100`
- `sinh(gl) ≈ exp(gl) / 2` при `Re(gl) > 100`

Это работает потому что `exp(-gl) / 2 → 0` при больших `Re(gl)`.
