# Упрощённый интерфейс редактирования кабелей (v1.4)

## Что изменилось

UI для редактирования параметров кабеля теперь содержит только:
1. **Тип кабеля** (выбор из списка предустановленных типов)
2. **Длина кабеля L** (в метрах, редактируемо)
3. **Статус спектра** (информационное сообщение о загруженных параметрах)

## Как это работает

### При выборе типа кабеля:
1. Система расчитывает **спектр α и β** для выбранного типа на диапазоне **0.01–1000 МГц** (200 точек)
2. Спектр **сохраняется в JSON** для повторного использования
3. Z₀ (волновое сопротивление) определяется на **эталонной частоте 24 МГц** (Z₀ примерно постоянна, не зависит от частоты)
4. UI отображает: **"✓ Завантажений"** (спектр загружен)

### При расчёте схемы (build_cascade):
1. Пользователь выбирает **частотный диапазон** (f_min, f_max, количество точек)
2. Система рассчитывает ABCD-матрицы для **каждой частоты в диапазоне**
3. Для каждой частоты:
   - **α** и **β** интерполируются из сохраённого спектра
   - **Z₀** интерполируется из спектра (примерно постоянна, но может немного варьироваться)
4. Используются **интерполированные значения**, не фиксированные

## Преимущества

✅ **Упрощённый интерфейс** — пользователь задаёт только тип и длину  
✅ **Точные расчёты** — параметры интерполируются для каждой частоты в выбранном диапазоне  
✅ **Нет ошибок ввода** — α, β, Z₀ не редактируются вручную  
✅ **Кэширование спектров** — спектры сохраняются в JSON, не пересчитываются  

## Что НЕ отображается в properties panel

❌ α (затухание) — зависит от частоты, будет интерполирована при расчёте  
❌ β (коэффициент фазы) — зависит от частоты, будет интерполирована при расчёте  

## Для пользовательских кабелей

Если выбран пункт "Користувацький":
- Можно вручную вводить α и β (в будущем добавить редактируемые поля)
- Параметры не будут интерполироваться из спектра
