# МБЕ / PLC — Объединённое приложение

## Краткое описание

**`unified_app.py`** — это объединённая программа, которая комбинирует:

1. **Расчёт параметров кабелей** (из `main.py`)
   - Вычисляет α (затухание), β (коэффициент фазы), Z₀ (волновой импеданс)
   - Для разных типов кабелей ШВВП и материалов проводников
   - Сохраняет результаты в базу данных `cable_params.json`

2. **Редактор сетевых схем** (из `schema.py`)
   - Визуальное построение топологии MBE сети
   - Выбор типа кабеля с **автоматическим заполнением параметров**
   - ABCD-анализ каскадной сети с поддержкой разветвлений

## Как пользоваться

### Шаг 1: Запуск
```bash
python unified_app.py
```

### Шаг 2: Построение схемы
1. Нажмите **"📡 TX"** → разместите на схеме (передатчик MBE)
2. Нажмите **"📺 RX"** → разместите (приёмник MBE)
3. Нажмите **"● Node"** → добавьте узлы разветвления (опционально)
4. Нажмите **"⊥ Load"** → добавьте нагрузки на ответвления (опционально)
5. Нажмите **"╌ Кабель"** → соедините элементы (сначала TX→Node→RX)

### Шаг 3: Выбор типа кабеля
Для каждого кабеля:
1. Щелкните на кабель (выделится на схеме)
2. На правой панели нажмите "КАБЕЛЬ"
3. В выпадающем списке выберите **"ШВВП 2×1.5"** (или другой тип)
4. Выберите материал: **"copper"** или **"aluminium"**
5. Параметры автоматически заполнятся! ✓

**Примечание:** Параметры рассчитываются на базовой частоте **24 МГц** (стандартная частота для MBE).

### Шаг 4: Расчёт сети
1. Установите частотный диапазон (например, 1 — 30 МГц)
2. Нажмите **"▶ РАСРАХУВАТИ"**
3. На графиках справа появятся:
   - **Верхний график** — Загасание [дБ] по частоте
   - **Нижний график** — Фаза [рад] по частоте

## Поддерживаемые типы кабелей

| Тип | Сечение | Дротов | GOST |
|-----|---------|--------|------|
| ШВВП 2×1.5 | 1.5 мм² | 30 | ГОСТ 1508 |
| ШВВП 2×2.5 | 2.5 мм² | 50 | ГОСТ 1508 |
| ШВВП 2×4   | 4.0 мм² | 80 | ГОСТ 1508 |

## Материалы проводников

- **Мідь (copper)** — ρ = 0.0175 Ом·мм²/м
- **Алюміній (aluminium)** — ρ = 0.0262 Ом·мм²/м

## База данных параметров

Файл **`cable_params.json`** автоматически создаётся и содержит кэш расчётов:

```json
{
  "ШВВП 2×1.5_copper_24MHz": {
    "alpha": 144.192,
    "beta": 1087.281,
    "Z0": 66.851,
    "frequency_mhz": 24.0,
    "cable_type": "ШВВП 2×1.5",
    "material": "copper"
  }
}
```

**Преимущества:**
- ⚡ Быстрая загрузка параметров (не нужно пересчитывать)
- 💾 Сохранение истории расчётов
- 🔄 Переиспользование параметров в разных проектах

## Технические параметры расчётов

### Первичные параметры (на км):
- **R₁** — погонное сопротивление [Ом/км]
- **L₁** — погонная индуктивность [Гн/км]
- **C₁** — погонная ёмкость [Ф/км]
- **G₁** — погонная проводимость [Сим/км]

### Вторичные параметры:
- **α** — коэффициент затухания [Нп/м] → [дБ/км]
- **β** — коэффициент фазы [рад/м]
- **Z₀** — волновой импеданс [Ом]
- **φ** — фаза волнового импеданса [рад]

### Формулы:
```
γ = α + j·β  (комплексная постоянная распространения)
Z₀ = √((R₁ + jωL₁)/(G₁ + jωC₁))
```

## Примеры использования

### Пример 1: Простая линия TX → RX

```
TX (Zs=75Ω) ──[ШВВП 2×1.5, L=30м]──► RX (ZL=75Ω)
```

1. Добавьте TX, RX
2. Соедините кабелем
3. Выберите "ШВВП 2×1.5" и "copper"
4. Нажмите "▶ РАСРАХУВАТИ"

### Пример 2: Разветвлённая сеть

```
TX ──[L1]──► Node1 ──[L2]──► Node2 ──[L3]──► RX
              │               │
             [L4]            [L5]
              │               │
              ▼               ▼
            Load1           Load2
```

1. Добавьте все элементы
2. Соедините как на схеме выше
3. Для каждого кабеля выберите тип
4. Программа автоматически пересчитает входные импедансы нагрузок
5. Нажмите "▶ РАСРАХУВАТИ"

## Часто задаваемые вопросы

**Q: Почему параметры рассчитываются на 24 МГц?**  
A: Это базовая подносящая частота стандарта MBE (ModBus Encoder). На этой частоте определяются основные характеристики кабельной системы.

**Q: Могу ли я добавить свой тип кабеля?**  
A: Да! Отредактируйте словарь `CABLE_TYPES` в классе `CableCalculator` (строка ~150).

**Q: Что если выберу "Користувацький"?**  
A: Вы сможете вручную ввести α, β, Z₀ вместо автоматического расчёта.

**Q: Как удалить кэш расчётов?**  
A: Удалите файл `cable_params.json` — он пересоздастся автоматически при следующих расчётах.
