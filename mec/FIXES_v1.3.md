## 🔧 Исправленные проблемы

### ✅ Проблема 1: RuntimeWarning: overflow в np.exp()

**Симптом:**
```
RuntimeWarning: overflow encountered in exp
RuntimeWarning: invalid value encountered in scalar divide
```

**Причина:** Функция `np.exp(gl)` переполняется при Re(gl) > 700

**Решение:** Добавлены три уровня защиты:
1. **Пороговая защита:** При Re(gl) > 700 вернуть 1e300 вместо попытки вычислить exp(gl)
2. **Проверка конечности:** Для Re(gl) в диапазоне 100-700 проверяем `np.isfinite(exp_gl)` перед делением
3. **Try-except блок:** На случай если exp() выбросит исключение

**Результат:** ✅ Нет RuntimeWarning, все значения валидные

### ✅ Проблема 2: Тип кабеля теряется при переключении

**Симптом:** Переключаешь кабель на другой и обратно — тип теряется из комбобокса

**Причина:** Функция `_on_cable_change()` получала ссылку на словарь edge, но из-за особенностей Python UI, эта ссылка могла указывать на старый объект. При обновлении панели создавался новый edge объект и старые изменения терялись.

**Решение:** Передавать **edge_id вместо edge**, и всегда получать актуальную ссылку из графа:

```python
def _on_cable_change(self, edge_id, v_cable):
    """При виборі типу кабелю"""
    # Получить актуальную ссылку!
    edge = self.graph.edges.get(edge_id)
    if edge is None:
        return
    
    edge['cable_type'] = v_cable.get()
    self.on_change()
```

**Изменения:**
- `_show_edge()`: Передает `eid` вместо `e`
- `_on_cable_change()`: Принимает `edge_id`, получает edge из графа
- `_on_material_change()`: Принимает `edge_id`, получает edge из графа

**Результат:** ✅ Тип кабеля надежно сохраняется при любых переключениях
