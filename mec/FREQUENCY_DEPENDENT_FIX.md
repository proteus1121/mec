# Исправление: Параметры кабеля зависят от ЧАСТОТНОГО ДИАПАЗОНА, а не от 24 МГц

## Проблема (v1.4)
Исходная реализация показывала параметры α, β, Z₀ **на фиксированной частоте 24 МГц**, но при расчёте схемы пользователь выбирает **произвольный частотный диапазон** (f_min, f_max, количество точек). Это несоответствие приводило к неправильным расчётам.

## Решение (v1.4.1)

### 1. **В properties panel (UI)**
Вместо показа фиксированных значений:
```
α = 144.190651 Np/m  (на 24 МГц)
β = 1087.301811 rad/m (на 24 МГц)
Z₀ = 66.8 Ω         (на 24 МГц)
```

Теперь показываем статус спектра:
```
Спектр параметрів: ✓ Завантажений
Примітка: Параметри інтерполюються для вибраного діапазону частот
```

### 2. **В edge (структура данных)**
- ❌ **Удалены**: `edge['alpha']`, `edge['beta']` (фиксированные значения)
- ✅ **Сохраняются**: `edge['cable_type']`, `edge['material']`, `edge['Z0']` (Z₀ как эталонное)
- ✅ **Спектр**: сохраняется в JSON для интерполяции

### 3. **В build_cascade (расчёт схемы)**
```python
for freq in freq_array:  # Пользователь выбирает диапазон
    for cable_edge in cables:
        # Интерполируем α, β, Z₀ для ТЕКУЩЕЙ частоты
        params = calc.get_params_at_freq(cable_type, material, freq)
        alpha = params['alpha']   # ← интерполировано для freq
        beta = params['beta']     # ← интерполировано для freq
        Z0 = params['Z0']        # ← интерполировано для freq
        
        # Используем интерполированные значения
        M = abcd_line(gamma, Z0, L)
```

## Преимущества

✅ **Точность**: расчёты учитывают частотную зависимость параметров  
✅ **Гибкость**: работает с любым выбранным диапазоном частот  
✅ **Логика**: UI не вводит в заблуждение фиксированными значениями  
✅ **Производительность**: спектры кэшируются в JSON, не пересчитываются
