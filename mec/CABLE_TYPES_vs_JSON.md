# 🔧 CABLE_TYPES vs cable_params.json

## Основная идея

В unified_app.py используются **ДВА разных хранилища данных** для разных целей:

---

## 1. `CABLE_TYPES` — Константы в коде (геометрия)

**Где находится:** `unified_app.py`, строка ~160 в классе `CableCalculator`

**Что это:**
- **Физические параметры** кабелей (размеры, количество проводников)
- **Неизменяемые константы**, которые характеризуют конструкцию кабеля
- **Входные данные** для расчётов параметров

**Пример данных:**
```python
CABLE_TYPES = {
    'ШВВП 2×1.5': {
        'n_wires':     30.0,      # количество дротов в жиле
        'd_wire_mm':   0.238,     # диаметр одного дрота [мм]
        'Liz1_mm':     0.5,       # толщина изоляции [мм]
        'd1_mm':       1.53,      # диаметр жилы [мм]
        'a1_mm':       2.53,      # расстояние между центрами [мм]
        'r1_mm':       0.765,     # радиус жилы [мм]
        'psi':         0.6,       # коэффициент ёмкости
    },
    # ... другие типы
}
```

**Когда используется:**
- При создании нового расчёта
- Как исходные данные для численного решения уравнений ТЛП
- Для вычисления первичных параметров (R₁, L₁, C₁, G₁)

**Может ли меняться:** 
- Только если отредактировать исходный код (добавить новый тип кабеля)
- Не меняется во время работы программы

---

## 2. `cable_params.json` — База данных кэша (результаты расчётов)

**Где находится:** `mec/cable_params.json` (файл на диске)

**Что это:**
- **Результаты расчётов**: α, β, Z₀ для конкретных комбинаций
- **Кэш** для избежания пересчёта одних и тех же параметров
- **Выходные данные** расчётных функций

**Пример данных:**
```json
{
  "ШВВП 2×1.5_copper_24MHz": {
    "alpha": 144.191599,        ← затухание [Нп/м]
    "beta": 1087.281047,        ← коэф. фазы [рад/м]
    "Z0": 66.85105381,          ← волновой импеданс [Ом]
    "frequency_mhz": 24.0,      ← на какой частоте рассчитано
    "cable_type": "ШВВП 2×1.5", ← какой тип кабеля
    "material": "copper"        ← какой материал
  }
}
```

**Когда используется:**
- При выборе типа кабеля в интерфейсе
- Чтобы не пересчитывать параметры заново
- Для быстрой загрузки уже известных значений

**Может ли меняться:**
- **ДА!** Создаётся автоматически при первом расчёте
- Растёт при расчёте новых комбинаций (кабель + материал)
- Можно удалить — пересчитается автоматически

## 📊 Таблица сравнения

| Аспект | CABLE_TYPES | cable_params.json |
|--------|------------|------------------|
| **Тип данных** | Физические параметры | Результаты расчётов |
| **Местоположение** | В памяти (константы в коде) | На диске (JSON файл) |
| **Примеры** | диаметр, кол-во проводов | α, β, Z₀ |
| **Изменяемость** | Только отредактировав код | Автоматически растёт |
| **Функция** | Входные данные | Кэш для ускорения |
| **Может быть удалён** | НЕТ (в коде) | ДА (пересчитается) |
| **Размер** | ~1 KB | ~5-50 KB (растёт) |

## ✏️ Как добавить новый тип кабеля

**Шаг 1:** Отредактируйте `CABLE_TYPES` в коде:

```python
'ШВВП 3×2.5': {              # Новый тип
    'n_wires':    60.0,
    'd_wire_mm':  0.238,
    'Liz1_mm':    0.5,
    'lengthk1_mm':7.0,
    'd1_mm':      2.1,       # Нужно знать физические параметры!
    'a1_mm':      3.5,
    'r1_mm':      1.05,
    'psi':        0.6,
},
```

**Шаг 2:** Перезагрузите программу

**Шаг 3:** При выборе нового типа произойдёт расчёт и сохранение в `cable_params.json`

---

## 🎯 Почему два хранилища?

1. **CABLE_TYPES** = базовая физика (не меняется)
   - Нужны для расчётов
   - Встроены в программу
   - Быстрый доступ (в памяти)

2. **cable_params.json** = результаты (растёт со временем)
   - Сохраняет историю расчётов
   - Позволяет не пересчитывать одно и то же
   - Можно обменяться между пользователями
   - Можно анализировать (какие кабели были использованы)

---

## 🔍 Итог

- **CABLE_TYPES** = "что такое кабель ШВВП 2×1.5?" (геометрия)
- **cable_params.json** = "какие параметры у этого кабеля на 24 МГц?" (результат)

Оба нужны для работы программы! 🎉
