## ❌ Проблема 1: Не сохранялся тип кабеля

### Что было
При смене типа кабеля в выпадающем списке (комбобокс) выбор **не сохранялся** в структуру данных кабеля. При повторном открытии кабеля вместо сохранённого типа показывался первый элемент списка.

### Почему произошла
В функции `_on_cable_change()` была пустая команда при выборе пользователя:
```python
if cable_type == 'Користувацький':
    return  # ← ничего не происходит!
```

Тип кабеля сохранялся **только** при смене материала, а не при смене типа.

### Как исправлено
✅ Добавлено сохранение типа **сразу при выборе**:
```python
edge['cable_type'] = cable_type  # ← ключевая строка
```

✅ Улучшена восстановление выбранного значения при открытии:
```python
current_cable = e.get('cable_type')
if not current_cable or current_cable not in cable_types:
    current_cable = cable_types[0]
v_cable = tk.StringVar(value=current_cable)
```

**Результат:** ✅ Тип кабеля теперь сохраняется при любой смене

## ❌ Проблема 2: Зачем два JSON?

### Что спрашивал пользователь
"Зачем два JSON? CABLE_TYPES и cable_params.json?"

### Уточнение
Это **НЕ два JSON файла**, а:

1. **CABLE_TYPES** — **константы в коде** (словарь Python с геометрическими параметрами кабелей)
   - Находится: в памяти программы (unified_app.py, строка ~160)
   - Содержит: диаметры, количество дротов, изоляцию и т.д.
   - Функция: входные данные для расчётов
   - Изменяемость: только редактированием кода

2. **cable_params.json** — **файл базы данных на диске**
   - Находится: `mec/cable_params.json` (реальный файл)
   - Содержит: результаты расчётов (α, β, Z₀)
   - Функция: кэш для избежания пересчёта
   - Изменяемость: растёт при каждом новом расчёте

**Аналогия:** CABLE_TYPES = "шпаргалка с размерами кабеля", cable_params.json = "тетрадь с уже решёнными задачами"

📄 Смотрите [CABLE_TYPES_vs_JSON.md](CABLE_TYPES_vs_JSON.md) для подробного объяснения

## ✅ Улучшение 1: Добавлено сохранение схемы

### Что добавлено
Две новые кнопки в интерфейсе:
- **"💾 Схема"** — сохранить текущую схему в JSON файл
- **"📂 Відкрити"** — загрузить схему из JSON файла

### Как работает
```
1. Нарисовали схему → выбрали типы кабелей → установили параметры
   ↓
2. Нажали "💾 Схема" → выбрали имя файла (например, "network1.json")
   ↓
3. ✓ Сохранилось: узлы, кабели, ТИПЫ кабелей, материалы, всё!
   ↓
4. Закрыли и снова открыли программу
   ↓
5. Нажали "📂 Відкрити" → выбрали "network1.json"
   ↓
6. ✓ Схема полностью восстановлена!
```

### Что сохраняется
В JSON файл попадает:
- Все узлы (TX, RX, Node, Load) с координатами и параметрами
- Все кабели с параметрами (**включая тип кабеля и материал**)
- Все расчётные параметры (α, β, Z₀)

**Пример сохранённого кабеля:**
```json
{
  "n1": 0, "n2": 1,
  "L": 30,
  "alpha": 144.191599,
  "beta": 1087.281047,
  "Z0": 66.85,
  "cable_type": "ШВВП 2×1.5",   ← сохраняется!
  "material": "copper",           ← сохраняется!
  "frequency_mhz": 24.0
}
```

**Результат:** ✅ Полная персистентность схемы между сеансами

## ✅ Улучшение 2: Лучше восстанавливаются сохранённые значения

### Что было
```python
v_cable = tk.StringVar(value=e.get('cable_type') or cable_types[0])
```
Просто, но может дать ошибку если тип кабеля был удалён из списка.

### Что стало
```python
current_cable = e.get('cable_type')
if not current_cable or current_cable not in cable_types:
    current_cable = cable_types[0]
v_cable = tk.StringVar(value=current_cable)
```
Проверяет валидность перед установкой.

**Результат:** ✅ Более надёжное восстановление состояния UI
